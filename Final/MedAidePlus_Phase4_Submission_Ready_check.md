**AREA OF WORK**

This project lies at the intersection of **Medical AI, Large Language
Models (LLMs), Multi-Agent Systems, Clinical Decision Support, and Safe
AI Deployment**. The core problem addressed is that real clinical
conversations are multi-dimensional: one patient query may require
triage reasoning, diagnosis-focused reasoning, medication safety checks,
and post-treatment monitoring guidance. A single static assistant often
fails to maintain quality across all these modes.

The work extends the MedAide architecture to a complete seven-module
system, MedAidePlus, adding persistent patient memory, hallucination
detection, adaptive complexity routing, and multi-turn intent tracking.
The project also contributes a controlled benchmark and ablation
pipeline for reproducible evaluation on local open-weight LLMs.

# ABSTRACT

This report presents **MedAidePlus**, an enhanced multi-agent medical AI
framework designed to improve reliability and clinical relevance of
responses generated by open-weight LLMs. MedAidePlus extends the
baseline MedAide system from three modules to seven modules by adding
persistent patient memory, hallucination-aware verification, adaptive
complexity routing, and multi-turn intent evolution tracking.

The central technical contribution is **category-aware specialist
routing**, which deterministically selects the clinically appropriate
specialist agent (Pre-Diagnosis, Diagnosis, Medication, Post-Diagnosis)
rather than relying only on content-level classification. A second major
contribution is **extractive primary-agent synthesis**, which preserves
specialist clinical vocabulary and structure by avoiding unnecessary
paraphrasing.

A 100-instance benchmark (25 instances per category) was created and
evaluated across three local LLMs (gemma3:4b, qwen3:8b,
phi4-reasoning:14b) using six metrics: BLEU-1, BLEU-2, METEOR, ROUGE-L,
BERTScore, and GLEU. MedAidePlus consistently outperformed MedAide
across all metric-model comparisons, demonstrating improved alignment
with clinically structured references.

The project also identifies two practical evaluation risks in medical AI
systems: specialist misrouting and stale persistent-memory contamination
across benchmark runs. Mitigations for both are implemented and
validated.

# 1. INTRODUCTION

Large language models are increasingly capable in medical question
answering, but real-world patient support requires more than generic
text generation. Clinical interactions involve urgency triage,
diagnostic reasoning, medication safety, and post-treatment follow-up.
These represent different reasoning patterns and should be handled by
distinct specialist policies.

The original MedAide architecture introduced a specialist-based design
but had limitations: simplistic routing, no robust memory isolation, no
calibrated hallucination verification, and limited conversation-state
handling. MedAidePlus addresses these limitations through a modular and
evaluable design.

1.1 Problem Statement

Single-agent or weakly-routed medical assistants may generate clinically
plausible yet structurally misaligned responses, especially when query
categories overlap in vocabulary (for example, diagnosis queries
containing symptom-heavy wording). This causes misrouting and degraded
quality.

1.2 Objectives

- Build an end-to-end seven-module medical AI pipeline.

- Improve specialist selection through category-aware routing.

- Add persistent memory with benchmark-safe isolation.

- Add hallucination-aware verification.

- Evaluate rigorously on multiple open-weight local LLMs.

1.3 Scope

This report focuses on text-based clinical consultation support and
benchmarked automatic metrics. It does not claim deployment as a
certified medical device and is intended for research and educational
use.

# 2. LITERATURE REVIEW / SURVEY

Medical LLM research has progressed through domain adaptation,
instruction tuning, and retrieval augmentation. Prior work such as
MedAide demonstrated that task decomposition with specialist agents
improves structure and interpretability over monolithic prompting.

Multi-agent reasoning frameworks (for example, ReAct and
Reflexion-inspired systems) motivate explicit role separation and
orchestration. However, in high-stakes domains, reliability concerns
remain: hallucination, inconsistent terminology, and unstable routing.

Relevant foundational works include:

- MedAide (multi-agent clinical architecture)

- BioBERT (biomedical embeddings)

- BM25 relevance modeling for factual support

- BLEU, ROUGE, METEOR, and BERTScore for text-quality evaluation

The key gap identified is not only model capability, but **system-level
alignment**: selecting the right specialist, preserving specialist
structure, and preventing benchmark confounders from persistent memory.

# 3. METHODOLOGY AND WORK DESCRIPTION

MedAidePlus is organized into seven modules executed in sequence:

- M1 AMQU: Adaptive Multi-shot Query Understanding

- M2 HDIO: Hierarchical Dual-level Intent Ontology

- M3 DMACN: Dynamic Multi-Agent Critic Network

- M4 PAHM: Persistent and Historical Memory

- M5 HDFG: Hallucination-aware Dual-verification Framework

- M6 AQCR: Adaptive Query Complexity Router

- M7 MIET: Multi-turn Intent Evolution Tracking

3.1 Core Design Decisions

- Deterministic specialist selection through category hints during
  evaluation.

- Extractive final-response synthesis from the primary specialist.

- Explicit patient-graph cleanup before benchmark runs.

- Deterministic generation setup (temperature = 0) for reproducibility.

3.2 Benchmark Setup Summary

- Total instances: 100

- Categories: Pre-Diagnosis (25), Diagnosis (25), Medication (25),
  Post-Diagnosis (25)

- Models: gemma3:4b, qwen3:8b, phi4-reasoning:14b

- Metrics: BLEU-1, BLEU-2, METEOR, ROUGE-L, BERTScore, GLEU

# 4. PROPOSED ALGORITHM

Algorithm: Category-Aware Specialist Inference Pipeline

**Input:** user query q, optional category_hint c, patient_id p,
conversation state s\
**Output:** clinically structured response r

1.  Decompose q into sub-queries and retrieve relevant medical evidence
    (M1).

2.  Compute intent/category scores from hierarchical ontology (M2).

3.  If category_hint c is available, override content-only routing with
    c.

4.  Select corresponding specialist agent from category-to-agent map.

5.  Load relevant patient history from PAHM for patient p (M4).

6.  Generate specialist response via selected agent (M3).

7.  Run hallucination support and uncertainty checks (M5).

8.  Route complexity metadata and maintain deterministic inference
    profile (M6).

9.  Update dialogue evolution state for session continuity (M7).

10. Return primary specialist response with validated structured
    sections.

4.1 Category to Specialist Mapping

- Pre-Diagnosis -\> PreDiagnosisAgent

- Diagnosis -\> DiagnosisAgent

- Medication -\> MedicationAgent

- Post-Diagnosis -\> PostDiagnosisAgent

# 5. PROPOSED FLOWCHART / BLOCK DIAGRAM

5.1 Conceptual Block Flow

User Query -\> M1 AMQU -\> M2 HDIO -\> (Category Hint + AQCR) -\>
Specialist Selection (M3) -\> Response Generation -\> M5 Hallucination
Verification -\> M4/M7 Memory Update -\> Final Response

5.2 Operational Notes

- For benchmark evaluation, category-aware routing is enforced to test
  specialist alignment.

- Patient graphs are reset before each benchmark run to avoid
  stale-memory contamination.

- Inference is deterministic to ensure stable comparison across systems.

# 8. RESULT ANALYSIS

8.1 Main Quantitative Results

MedAidePlus achieves universal gains over the MedAide baseline across
all six metrics and all three evaluated LLMs.

Table 1: Main Comparison (Averages)

| System      | BLEU-1 | BLEU-2 | METEOR | ROUGE-L | BERTScore |  GLEU |
|-------------|-------:|-------:|-------:|--------:|----------:|------:|
| MedAide     |  22.14 |   9.52 |  16.34 |   14.81 |     55.38 |  7.23 |
| MedAidePlus |  25.87 |  12.39 |  19.83 |   17.59 |     60.64 |  9.02 |
| Delta       |  +3.73 |  +2.87 |  +3.49 |   +2.78 |     +5.26 | +1.79 |

8.2 Category-Level Observations

- Medication and Diagnosis categories show strongest gains due to
  specialist-agent alignment.

- Post-Diagnosis also improves consistently with routing and structure
  preservation.

- Pre-Diagnosis differences are smaller because both systems often use
  similar specialist behavior there.

8.3 Ablation Insights

Ablation on qwen3:8b shows:

- Adding new modules without routing correction can reduce score.

- Extractive synthesis recovers quality by preserving terminology.

- Category-aware routing yields the largest positive jump.

- Memory isolation removes hidden benchmark confounds and stabilizes
  final results.

8.4 Interpretation

Performance improvement is driven primarily by **system design
correctness** (who answers, and how final wording is preserved), not by
increasing model size alone.

# 9. CONCLUSION AND FUTURE SCOPE

This project demonstrates that MedAidePlus, a seven-module medical
multi-agent framework, significantly improves benchmark performance over
MedAide through category-aware specialist routing and extractive
synthesis. The architecture also introduces practical safeguards for
memory contamination and hallucination-aware checks.

9.1 Major Contributions

- Seven-module clinical AI orchestration pipeline.

- Deterministic category-to-specialist mapping for correct domain
  handling.

- Persistent memory with benchmark-safe isolation.

- Reproducible multi-model evaluation with comprehensive metrics.

9.2 Future Scope

- Train HDIO with larger clinically annotated datasets (for production
  routing without hints).

- Add human clinical evaluation and inter-rater reliability study.

- Expand benchmark scale to 500+ cases and include longitudinal
  sessions.

- Explore consensus synthesis for genuinely multi-specialist complex
  cases.

- Incorporate stronger privacy and audit controls for deployment-grade
  settings.
